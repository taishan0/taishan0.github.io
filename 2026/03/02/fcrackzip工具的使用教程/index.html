二、核心参数（记这几个就够用）

表格

参数 作用 通俗理解
-b 暴力破解模式 挨个试所有可能的密码组合
-D 字典破解模式 用现成的密码字典文件去试
-c 指定字符集 限定密码用什么字符（数字 / 大小写 / 符号）
-l 密码长度范围 比如 -l 4-6 表示只试 4~6 位密码
-u 验证解压 自动用 unzip 验证密码，只输出正确结果
-p 字典路径 字典破解时，指定字典文件位置
-v 详细输出 显示正在试的密码，方便看进度

字符集常用写法（-c 后面）：

  • 1 = 只试数字（0-9）
  • a = 只试小写字母（a-z）
  • A = 只试大写字母（A-Z）
  • ! = 常用特殊符号（!:$%&/()=?{[]}+*~#）
  • 组合：aA1! = 大小写 + 数字 + 符号

五、常见问题

  1. 提示找不到文件：检查压缩包路径是否正确，可用 ls 看当前目录文件。
  2. 破解极慢：暴力破解时，字符集 / 长度范围别太大（比如 -l 1-10 会很慢），优先用字典破解。
  3. 提示 “不是 zip 加密文件”：可能是伪加密，可用 binwalk 等工具分析。

六、一句话总结

  • 短密码、已知规则 → 用 暴力破解（-b -c -l -u）
  • 长密码、CTF 场景 → 用 字典破解（-D -p 字典 -u）
